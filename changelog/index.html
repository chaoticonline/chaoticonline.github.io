Current Version

Commited in 02/07/2018 at 10:30:00

Added

Structure-related Changes

  • Added the HTML entity &nbsp; to all almost all empty tags preventing them to be really empty and therefore considered as removal candidates by HTML Minifiers

  • Added a different error message to Card Search, allowing one of the two fatal errors that could yield no results while using the Card Search to be better distinguished

  • Added an entry to the .gitignore wrongly removed in previous commits

  • Added the SiteNavigationElement Structured Data pointing to the Navigation Menu as another SEO improvement

Video Pages-related changes
  • Published Episode #15, Season #01 with Brazilian Portuguese audio
Hugo Templates-related Changes

  • Added/Adjusted several of the Bootstrap Grid Classes in all Template Files to provide maximum responsiveness in different devices and/or viewport resolutions

  • Added the inLanguage property to the Schema.org Linked Data Structured Data

  • Added a <wrapper> element to encapsulates the whole content of the site’s pages, as part of the fix for the footer covering part of the main content. The technique was borrowed from this Codeply and, although it works, the presence of the <wrapper> tag in there makes it semantically invalid

CSS-related changes

  • Added Github and Hugo icons, previously added to the UI’s SVG file, to the new footer

  • Added more Media Queries to side the better responsiveness created with the Bootstrap’s Grid classes changes/additions

Public Content related changes

  • Added appreciation to the VERO Team in the Disclaimer Page

  • Added this page with information for geeks (:p) with all version changes

Removed

Structure-related Changes
  • Removed the i18n entry for the Patron social link. Not only because it’s long hidden but also because the new social links markup (see below) doesn’t use any text
Hugo Templates-related Changes

  • Removed zoom-level restrictions

  • Removed some of Hugo’s absURL function usage in favour of Relative URLs

  • Removed unnecessary indentation levels of all Partial Templates, after all, they’ll be used in a composite format but edited individually

CSS-related changes

  • Removed the top-margin used on Flag Icons because the new Navigation Bar has better alignment thanks to the Flexbox used

  • With all new features and changes, A LOT of rules aren’t needed anymore and around 300 lines have been removed

Changed

Structure-related Changes
  • Changed some of the unnecessarily Absolute URL to Relative ones
Hugo Templates-related Changes

  • Moved the Schema.org Structured Data for Organization to the top, in a more hierarchical way

  • Moved the left-hand <aside> used for social links out of the main content, as it should be

  • Changed the main content wrapper tag from <div> to <main>

  • Changed the Navigation Menu Partial Template with a cleaner markup that requires almost no CSS micro-adjustments and brings better mobile responsiveness as well

  • Update the Footer Partial Template with a new and more responsive version in order to remove a mobile-specific alternative version

  • Modified the Big Picture markup in order to remove a mobile-specific alternative version

  • Changed the Social Links markup with a more simpler one. Although not fancier, that opens/closes on hover, this version doesn’t need any hacks to accommodate different lengths of text — i.e. the now hidden Patreon label

  • Modified the Cards’ Images markup for better vertical alignment -AND- to finally remove the long used workaround to deal with the “white corner fix” on Google Chrome

  • Updated all Section & Article Titles markup in order to make them more concise -AND- to remove a mobile-specific alternative version

  • With the removal of the left-hand <aside> from the main content, the column width for larger devices (width >= 996px) has been increased

  • Changed the CSS class used in one of the two possible fatal errors that could yield no results while using the Card Search

CSS-related changes

  • The new Navigation Menu rendered the Flyout Content — the box that opens on hover — of Languages Menu too small, so top and bottom padding has been added to them

  • With the improvements made to the Cards Images’ vertical alignment the Flavour Texts styles, in the form of Quotes, their inner and sibling components elements (i.e. ToC Tree) had to be adjusted as well because some Articles, about some Cards, have them right below a Card Image — i.e: Najarin

  • Adjusted the top-margin of the Loading Icons

  • Adjusted margins of Server Selection message and available Providers wrapper container

  • All rules needed for the other changes/additions described in this revision have been added, changed or removed

  • Reordered some classes in a more pseudo-hierarchical format

Public Content related changes

  • Applied grammar corrections to the Brazilian Portuguese version of the following Articles/Pages: Maxxor, Najarin, Blazier, Heptadd, Takinom, Nauthilax, H’earring, Chaor, Malvadine, Glacier Plains, UnderWorld City, Crystal Cave, Riverlands and Disclaimer and Terms Pages

  • Applied grammar corrections to the English version of Disclaimer and Terms Pages

Fixed

Structure-related Changes
  • Fixed an issue with Cards Search that was preventing it to work online, specifically because of HTML Minifiers removing empty elements that would receive content dynamically
Hugo Templates-related Changes
  • Fixed a long term mobile issue in which the site footer was covering the bottom part of the content

Miscellaneous Changes

  • Updated the UI’s SVG icons file, reordering some definitions in a more meaningful order

  • Removed PNG files of Video Providers as they’re not used anymore

  • Updated minified JS and CSS

Removed

Structure-related Changes

  • Removed the FrontMatter Parameter card and subtext, as they’re now handled by the picture Parameter and the “Quote” Shortcode, respectively

  • Removed the heading title “Abilities”, previously added under “Game”, as it was only cluttering the ToC Tree

  • Removed the FrontMatter Parameter languageMirrors as a sub-item of details, after all, it’s more related to the Content File which defines the structure rather than the Article/Video Page themselves

  • Removed the runtime script from Card Search as a FrontMatter Parameter as this is no longer an option, adding it directly in the script

Articles-related changes
  • Removed the main Article Name from their variants — e.g. instead of “Chaor — The Fierce” now only “The Fierce” is displayed — cleaning up the ToC Tree
Video Pages-related changes
  • Removed all Vimeo mirrors of all Episodes dubbed in Brazilian Portuguese
Hugo Templates-related Changes

  • With the changes made to the Flavour Texts in the form of Quotes (see below), the “Quote” Partial Template became obsolete and, therefore, has been removed

  • Removed useless conditionals from “Trivia” Template as they don’t have that many information like regular Episodes

  • Removed the Runtime Scripts feature from Footer Partial because while it could look interesting and organized, in fact, hard to maintain and, possibly, a security breach

Changed

Structure-related Changes

  • Changed the FrontMatter Parameter thumbnail to picture, after all, it handles the Big Picture more than the Articles’ Thumbnails from Listing Pages

  • Changed the FrontMatter Parameter group to section, because even though it doesn’t affect Hugo’s .Section variable it’s the most appropriate term to be used

  • Renamed the FrontMatter Parameter languageMirrors to languages, which is cleaner and matches its new concept better

  • Changed the way the FrontMatter Parameters eyes, hair and skin, all grouped under the traits Parameter, are handled. Now instead of having multiple values manually separated by a slash in a string, they’re all an indexed array automatically rendered as a list, like other Parameters that accept multiple values

  • Changed the way custom CSS styles are defined for Articles’ Pictures. Instead of all of them in a single line, like an inline declaration, now they’re defined in an associative array of two indices, rule and value. With this change, we can filter out rules that could cause unwanted behaviours, especially on mobile (i.e. height and max-height)

  • Reflected the FrontMatter Parameters renaming mentioned above in all Content Files

  • Migrated all Images’ Credits from the Disclaimer Page to each and every Article, Big Pictures and Card Arts

  • With the removal of the “Abilities” heading title mentioned above, all Cards’ variations are now one level deeper in the ToC Tree hierarchy, falling under “Game”

  • Renamed Content File of Dranakis Threshold to match the naming convention of all the others. By all means this represents a file removal and addition

Articles-related changes

  • Reviewed all Markdown links to add a slash (/) in the end of each one of them

  • Changed the skin entry under the FrontMatter Parameter trait of Blazier for Brazilian Portuguese translation, matching the “Sunflower Gold” of the English version

  • Changed the interpretation of the Mugicians expression in all Articles. Now they’re described only as “Mugicians” instead of “Mugicians’ Counters”, representing them as the actual spirit forms of the Mugic creators of the past, co-existing with the Creatures to power their Mugics

  • Typo, grammar and wording corrections and additions for the following Articles: Maxxor, Najarin, Blazier, Heptadd, Takinom, Nauthilax, H’earring (Brazilian Portuguese only), Chaor, Malvadine, Glacier Plains, Crystal Cave, Lake Ken-I-Po, Riverlands and UnderWorld City
