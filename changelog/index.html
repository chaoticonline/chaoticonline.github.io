Current Version

Committed at 08/27/2018 at 15:45

Added

Structure-related changes

  • Added i18n entries to describe Locations in the OverWorld and UnderWorld in order to make it easier for the Templating logics to build i18n keys for them

  • Added Hugo’s Data Files to all existing Creatures published so far, with their attributes, traits, episodes and voice actors

  • With the Content Files Componentization applied to the published Articles about Locations (see below), new i18n entries have been added for each one of them

Hugo Templates-related changes

  • Added HTML attributes width and height to <img/> tags of “WIP” default images

  • Added a conditional route on Listing Template to consider cases in which the Data File entry picture of the Card being listed hasn’t even been defined in order to add the “WIP” picture instead of leaving its corresponding Bootstrap Card without an <img/> tag

  • Modified the markup of Navigation Bar to accommodate the new Site Logo \o/

Public Content-related changes

  • Added one more Breadcrumb Trail on Cards’ variants Pages pointing to their main versions to reinforce the relationship between them

  • Added missing Breadcrumb Trail Videos from Trivias Pages

  • Added a new Content File about Ulmar’s Lab

  • Added list of Episodes in which Vidav appears as well as his Voice Actor in both English and Brazilian Portuguese

  • Added missing weight FrontMatter Parameter to the published Article about Bodal, so it can appear at the offset it should (English only)

  • Added missing weight for the published Article about Lava Pond has been added, so it can appear at the offset it should (Brazilian Portuguese only)

  • Added missing keyword for Thonder in Episode #12, Season #02

  • While adding Hugo’s Data Files, Dractyl’s Big Picture credits has been added (Brazilian Portuguese only)

  • While adding Hugo’s Data Files, Heptadd’s Crown rarity has been added

  • While adding Hugo’s Data Files, Tharax’s Disciplines have been added based on data from Episode #12, Season #01

  • Published the Episode #19, Season #01 with Brazilian Portuguese audio

CSS-related changes
  • Added a few rules as counterpart of the new Site Logo \o/

Removed

Structure-related changes

  • Removed URL aliases pointing to anchor/local links from the published Articles about Bodal, Flamedrill Researcher and Takinom, the Shadowknight as they’re not that important

  • With the officialdom of the Content File Componentization feature (see below), the temporary Template Files used during the BETA period have been removed

  • Removed Content Files (and with them, the Pages) about Growlitzer (Battlegear) and Danian Hall (Location) since they’re not real Cards and have very little information to even consider becoming an Article

  • With the addition of Hugo’s Data Files for all Creatures and Locations, the corresponding FrontMatter Parameters have been removed from all Content Files

Public Content-related changes

  • Removed all placeholder paragraphs, of all Content Files

  • While adding Hugo’s Data Files, all Big Pictures credits referring to HotShots VFX had their links removed since their Facebook Page doesn’t exist anymore

CSS-related changes
  • Removed the negative margin fo unordered lists (<ul>) to prevent vertically aligned text to be come misaligned, especially with Flexbox

Changed

Structure-related changes

  • Officiated the Content File Componentization feature previously implemented as BETA feature only in the published Articles about Takinom and Bodal

  • Changed URLs of the componentized Articles about Bodal, Flamedrill Researcher and Takinom, the Shadowknight to become a sub-division of their main Cards. While this change might not seem much for Creatures with their unique names, this reduces the chances of conflicts for other Card Types

  • Changed i18n entries of the Brazilian Portuguese URLs of the componentized Articles about Bodal, Flamedrill Researcher and Takinom, the Shadowknight to follow the naming structure of the mains versions (i.e. prefixing them with /pt). While this change might not seem much for Creatures with their unique names, this reduces the chances of conflicts for other Card Types, especially in other languages

  • Renamed the Brazilian Portuguese version of Mipedim Oasis to *Oásis de Mipedim*, as it sounds more lexically correct

  • With the change above, the corresponding i18n entries have been adjusted as well

  • Modified the directory structure of Hugo’s Data Files for Creatures with variations to follow the same structure of /perim

Hugo Templates-related changes

  • Moved the logic of Advertising Positioning in Listing Templates all the way to the top to make it easier to read given the number of nested blocks

  • With the removal of FrontMatter Parameters in favour of Hugo’s Data Files, the Partial Templates for Meta Tags have been updated in order to reflect the changes made, therefore, returning the tags temporarily removed in the previous revision

Public Content-related changes

  • Applied Content File Componentization to all published Articles about Locations because they always used the Card Shortcode instead of Picture like Creatures and, therefore, with the officialdom of the feature previously implemented as BETA, they would stop working

  • Translated the physical traits height and weight of Bodal, Flamedrill Researcher to, respectively, meters and kilos in the Brazilian Portuguese version of the published componentized Article

  • Fixed all reminiscent occurrences of English’s possessive pronouns in the Brazilian Portuguese version of the site

  • Uniformed all measuring units to NOT have the period after the units’ abbreviation

  • Uniformed all Cards’ variations to have a line-break before their “subheading”, like their Cards

  • With the directory structure modification of Hugo’s Data Files (see above), the Include Shortcode of the published Pages about Takinom and Bodal, included within their main Articles, were updated accordingly

  • Updated Changelog Page

Fixed

Structure-related changes

  • Fixed Content File structure of the Article about Symphonic Shield missing the <!--more--> special markup

  • Fixed several issues related to Structured Data JSON+LD created by a typo while implementing the new method for retrieving height and width of images

Public Content-related changes

  • Fixed letter-case of Whirling Wail (Attack) (English only)

  • Fixed list of Episodes in which Qwun and Antidaeon appear

  • Fixed Brazilian Portuguese Page about Cerbie whose link to the English version was wrongly pointing to Borth-Majar

  • While adding Hugo’s Data Files, Blazier’s physical traits skin and hair colours have been fixed (Brazilian Portuguese only)

  • While adding Hugo’s Data Files, Odu-Bathax’s Big Picture credits URL has been fixed as it was wrongly pointing to the same of Skartalas

  • While adding Hugo’s Data Files, Nauthilax’s physical trait height has been fixed (Brazilian Portuguese only)

  • While implementing the new FrontMatter Parameter data to associate a Content File with a Hugo’s Data File, the URL of the Page about Dractyl Scales (Battlegear) has been fixed (Brazilian Portuguese only)

  • Fixed a Markdown syntax for links used in the published Article about Najarin that weren’t properly interpreted and converted to HTML anchors

Image-related changes

  • Added a new image as Site Logo \o/

Miscellaneous changes

Added

Structure-related changes

  • Returned the Ribbons with Episodes’ numbers in their Thumbnails that was accidentally removed when the FrontMatter Parameters Structure changed a few revisions ago

  • Added missing keyword for Bodal in Episode #08, Season #02

  • Added missing Taxonomy Term type for Song of Futuresight (Brazilian Portuguese only)

Removed

Public Content-related changes

  • Removed the FrontMatter Parameters height and width from a few Content Files as these values come automatically from now on (see below)

  • Removed all occurrences of attribute class from Cards Shortcode, since it has no effect anymore, from the following published Articles: Maxxor, Najarin, Blazier, Heptadd, Tartarek, Chaor, H’earring, Nauthilax, Takinom, Ulmar, Malvadine, Zhade and Lake Ken-I-Po

  • Even though the Content File Componentization has been implemented as BETA feature in the publichsed Article about Takinom (and Bodal), all FrontMatter Parameters’ comments related to the removed Parameters have been removed

  • Removed one of the Deferred Callbacks from the main Cards Search Tool script exclusively to show the Loading Icon after resetting all UI styles as the benefits offered by this approach were minimal

Changed

Structure-related changes

  • Implemented a feature that’ll allow Creatures’ physical traits from FrontMatter Parameters to Hugo’s Data Files, allowing better organization and readability but, most important, the possibility of having Cards’ Variations’ Details present structurally in the markup, allowing them to be indexed by Cards Search Tool. For now, it’s only implemented in the Articles about Takinom and Bodal as a BETA feature

  • Implemented a feature that’ll allow the list of Episodes in which a Creature/Location appears from FrontMatter Parameters to Hugo’s Data Files, allowing better organization and readability but, most important, auto-linking. For now, it’s only implemented in the Article about Takinom and Bodal as a BETA feature

  • Implemented a feature that’ll allow identified Voice Actors of at least one of the languages of the site from FrontMatter Parameters to Data Files, allowing better organization and readability. For now, it’s only implemented in the Article about Takinom and Bodal as a BETA feature

Hugo Templates-related changes

  • Changed the way HTML attributes height and width are added to the <img/> tags. Now instead of reading a FrontMatter Parameter, Hugo’s Image Metadata Reader function imageConfig is used instead

  • Likewise the change above, the height and width values defined in JSON+LD Structured Data and <meta/> tags were changed using the same approach described above

  • Improved the Include Shortcode that was halted due hard times to find specific information on how to create componentizable Content Files with Hugo

Public Content-related changes

  • Typo, grammar and wording corrections and additions for the following Articles/Pages: Maxxor, Heptadd, Tartarek, H’earring, Nauthilax and Takinom

  • Renamed the HTML markup of Cards Search Tool to use the Parser name for All Cards as value of the Card Type dropdown

  • With the change above, the Tool’s main script, Search.js, has been adjusted to only instantiate the parser for All Cards when and if needed, instead of instantiating it by default and then overwrite it with a different one, thus improving performance

  • Updated Cards Search Tool to take into account the Content File Componentization implemented as a BETA feature in the Articles about Takinom and Bodal

  • Improved the Cards Search Tool so now it doesn’t need a “Swap Area” to alphabetically order the results by anymore

  • Improved the Cards Search Tool main script to not look for a specific selector multiple times by caching it, thus slightly improving performance

  • Improved the Cards Search Tool Filter.js script to not require Filter::filter() to be manually invoked in the main script by wrapping both actions, intersecting and retrieving filtered results, in a new method Filter::getFilteredData()

  • Improved the Cards Search Tool Filter.js script to ignore punctuation when sorting the filtered ResultSet. With this, Cards like Raznus, Ambassador to the Danians will stay among the other Cards starting with R instead of going to the end of the list

  • Modified all Data Parsers of Cards Search Tool so they can prepare the HTML element where the parsed data would go before actually parsing them. This improved the overall performance by removing a condition previously executed within a loop <_<

  • Updated Changelog Page

  • Fixed rarity of Skeletal Springs, wrongly defined as Common

  • Fixed rarity of Narfall, wrongly defined as Common (Brazilian Portuguese only)

  • Fixed rarity of Phelphor, wrongly defined as Rare

  • Fixed the i18n entries about Ørth, the Empowerer and his Content Files to fully describe his name instead of just Ørth

  • Fixed an issue with Cards Search Tool that was causing the search results when querying All Cards immediately after searching by a specific Card Type. In this scenario, the first results (by Type) were kept in the dynamically created table and all other results appended to them, messing with the sorting. Even though this is now fixed, in some locales this may give the impression of not fixed. For example: Most of the times, Aa’une is the first Card to be listed because of the two consecutive A in his name, but in Brazilian Portuguese for example, because the article the translates into a, the space after it in te translated version of Cards like The Passage (both versions) make these two Cards to appear before Aa’une instead

Structure-related changes

  • Changed the FrontMatter Parameter thumbnail to picture, after all, it handles the Big Picture more than the Articles’ Thumbnails from Listing Pages

  • Changed the FrontMatter Parameter group to section, because even though it doesn’t affect Hugo’s .Section variable it’s the most appropriate term to be used

  • Renamed the FrontMatter Parameter languageMirrors to languages, which is cleaner and matches its new concept better

  • Changed the way the FrontMatter Parameters eyes, hair and skin, all grouped under the traits Parameter, are handled. Now instead of having multiple values manually separated by a slash in a string, they’re all an indexed array automatically rendered as a list, like other Parameters that accept multiple values

  • Changed the way custom CSS styles are defined for Articles’ Pictures. Instead of all of them in a single line, like an inline declaration, now they’re defined in an associative array of two indices, rule and value. With this change, we can filter out rules that could cause unwanted behaviours, especially on mobile (i.e. height and max-height)
