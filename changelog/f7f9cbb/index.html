Chaotic Changelog rev. f7f9cbb

Changelog

  rev. f7f9cbb

 

Added

Structure-related changes
  • Added Content Files, Data Files and i18n entries for the following Pages as a counterpart of the references made in the Article about Dractyl published in this revision: Alacritous Force, Frost Blight and Power Pulse (Attacks) and Dread Tread, Elixir of Tenacity and Obliterators (Battlegears)
Hugo Templates-related changes

  • When using Hugo’s Content Minifier to optimize the output of all HTML files, we realized that all double quotes around the values of HTML attributes were also removed. While browsers understand this as valid, for some reason, this yielded several “unsecured domain” errors while making the required AJAX calls of the Card’s Search Tool, regardless of the fact of both the Requester (Tool’s Page) and the requested (each Page fitting the query filter, if any) be in the same domain and under SSL. In order to prevent that to happen, we created a new Shortcode called href that simply runs Hugo’s absURL function over a given argument. This would make our form action /tags/ to become https://www.chaoticonline.ml/tags/

  • Added a Partial Template to use Facebook Page Plugin, on hold until further testing

Public Content-related changes
  • Published Article about Dractyl
CSS-related changes

  • Added some spacing to the left of the Navigation Menu so it doesn’t get too close from Site Logo

  • Added a bottom-spacing to the fallback images so they don’t get too close from the Facebook Comments Plugin

  • Added some bottom-spacing for the Information Box of Location Pages

Removed

CSS-related changes

  • Removed the restriction of maximum width for unstyled lists. This way, items inside the Information Box have a bit more space for their content

  • Removed unused and unnecessary rules as well as some selectors commented-out for some time already

Changed

Structure-related changes

  • Changed the entry in .gitignore file that was preventing everything under Hugo’s Asset Minification directory (/assets) to be added to the repository, including the source files <_<. Now only Babel’s compiled scripts will be ignored, as they’re always regenerated when running the transcompiler

  • Changed the Brazilian Portuguese version of the Battlegear Orchis Undin to Orquídea D’água since it makes more sense given its Card Art and the fact that Undin refers to Undine, the mythological elemental being related to water

  • Modified, once again, the <urlset> attributes of all Card Types XML Sitemaps in a last attempt to have Google reading them

  • Updated Facebook SDK version to 3.1

Hugo Templates-related changes

  • Changed the condition to include the Bootstrap Column for the sidebar in the Footer Partial Template. Now we add the column regardless of the advertisable condition and restrict only the advertisement block to advertisable Pages. This will allow us to, in the future, add the newly created Partial Template for Facebook Pages Plugin regardless the Parametrizable condition as we’ll always have the layout column

  • Uniformed the HTML structure of the Loading Icon used in the Card’s Search Tool and in the Videos Pages by defining an id for it

Public Content-related changes

  • With the addition of the new href Shortcode, the Cards’ Search Tool’s Page have been updated to use it

  • With the renaming of the Brazilian Portuguese version of Orchis Undin, all occurrences of the old name have been updated throughout the site

  • Expanded the meaning of Tyrian purple with additional information in the published Articles about Takinom and Dractyl

  • Updated Changelog Page

CSS-related changes

  • Improved the vertical alignment of table-cells by overwriting Bootstrap’s default value so, now, they all align to the middle

  • With the uniformization of the HTML structure of the Loading Icon (see above), the corresponding CSS rules have been adjusted

  • Fixed an issue with the Loading Icon not spinning in Mozilla Firefox

  • Rescaled the Loading Icon, reducing its size

  • With the scale reduction above, the margins had to be changed as well

  • With the scale reduction above, the animation became a little too slow, so we speed it up by 500ms

  • Scaled up Site Logo a little bit and adjusted its margins for better horizontal alignment

  • Changed the way Location Cards have their dimensions reduced by rescaling them instead of forcing a smaller width

  • As a consequence of the change above, the images got larger, overlapping the Credits, so we adjusted their positioning

  • Reduced the top-margin of Information Box and increased the bottom-spacing

  • Because unstyled lists now have more space for their content, sometimes, it may seem a little “polluted”, so we added a small padding between each list-item to ease-out the visual aspect

  • When a bug with Facebook Comments Plugin made its comment box get shorter than it should even though it has been explicitly configured to be fluid, we tried to workaround the situation by forcing a maximum width. This didn’t solve the issue because the Plugin has a monstrous structure inside an <iframe>, which, for security reasons, can’t be manipulated unless on the same domain. Even though the issue seems to be resolved by Facebook Team, we’re keeping these modifications, just in case

  • With the modifications in Information Box margins, the Flavour Text element (a.k.a. “Quote”) element had to be adjusted too (again)

  • Reworked the structure of Loading Icon animations. Now instead of having the same properties for all animation steps in both regular and vendor-prefixed definitions, those targeting WebKit-based browsers (Chrome, Safari…) will only receive vendor-prefixed rules while regular definitions won’t, slightly reducing the file size

Fixed

Hugo Templates-related changes

  • Fixed an issue with the SEO Partial Template for Languages that was creating invalid entries when there was no FrontMatter Parameter language. Previously, it used to pass a manually defined /<LANG> string to Hugo’s absURL. In cases like of the homepage, even though the resulting /en would redirect correctly to /, that wouldn’t be valid as for the Alternate Languages concept

  • Fixed an issue with the SEO Partial Template for OpenGraph for Videos that used to set the Voice Actor name as the value of the video:actor property while the correct would point it to an URL with Actors’ Profile.

Public Content-related changes
  • Fixed the capitalization of Jagged Pillar (English only)
CSS-related changes
  • Fixed, with a palliative solution, an issue with Google Chrome showing a thin white margin in some of the thumbnails

Image-related changes

  • Replaced the Big Picture of Dractyl with a version of better quality of his original form, instead of the sinister, unused version

  • Added the following Big Pictures referenced in the Article about Dractyl published in this revision: Frost Blight and Power Pulse (Attacks) and Elixir of Tenacity (Battlegear)

  • Added a Card Image for Dractyl, as a counterpart for the Article published in this revision

Miscellaneous changes

  • Updated Sitemap Files for Attacks, Battlegears, Videos and Index

  • Updated minified CSS and JS
