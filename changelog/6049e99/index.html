Added

Structure-related changes

  • After a revision made to the FrontMatter Parameters infrastructure, the Parameters listed and comentable which, respectively, prevents an Article to be listed and allows it to be discussed through Facebook Comments Plugin, have been added, both set as false, in all published componentizable Articles of Locations variants

  • After a revision made to the FrontMatter Parameters infrastructure, all Pages related to a top-level in Navigation Menu (i.e. Perim) received in their _index.<LANG>.md files a new Parameter named category

  • After a revision made to the FrontMatter Parameters infrastructure, all Pages related to a sub-level in the Navigation Menu (i.e. Attacks) received in their /<SUBFOLDER>/_index.<LANG>.md files a new Parameter named category, overwriting their parent’s -AND- an additional Parameter named subcategory with the same effect of the now removed group

  • After a revision made to the FrontMatter Parameters infrastructure, all Pages related to either a top-level or a sub-level in the Navigation Menu received in their _index.<LANG>.md files the following Parameters:

    • commentable, defining whether or not the both the Listing Page and their children can be discussed through Facebook Comments Plugin
    • analytics, defining whether or not the both the Listing Page and their children will receive Google Analytics’ data-mining code
    • advertisable, defining whether or not the both the Listing Page and their children will receive Advertisements blocks, on the sidebar (visible on Desktop version only) and among the results (visible on mobile devices only) Theoretically, these Parameters should be added only in the topmost level Pages (i.e. /perim/_index.<LANG>.md), but their availability through Hugo’s .Params variable is too strict, with Pages under a sectioned Parent (i.e. /creatures) having their Parameters available through .CurrentSection.Params instead. This complicates the Template Development as we have to constantly switch between these aliases

  • Added two i18n entries to distinguish the Follow from the Share intentions in the revamped implementation of Social Links Template

  • Added a new Partial Template for Schema.org’s SiteNavigationElement to be rendered together in main the SEO Partial Template

  • Added a new Partial Template for Schema.org’s BreadcrumbList to be rendered together in main the SEO Partial Template

  • Added an instruction in the config.yaml file to automatically remove accents from paths, which allowed a massive reduction in the revamped implementation of SEO for Videos as we don’t need to remove them by hand anymore

  • Added an i18n identifier for Terms of Use, allowing the SEO Partial Template for Article to find a match for that Page as well

Hugo Templates-related changes
  • Added Properties isPartOf and mainEntityOfPage in the Partial Template for Lists
Public Content-related changes

  • Added missing Taxonomy Term type to Mipedim Oasis Page (English only)

  • Added missing Taxonomy Term initiative to Lava Pond and Carnival of Confusion Pages (Brazilian Portuguese only)

  • Added a reduced version of Cards’ Search Page intro as its summary, this way the SEO implementation has something better to work with

  • Published the Episode #22, Season #01 with Brazilian Portuguese audio

Removed

Structure-related changes

  • With a revision made to the FrontMatter Parameters infrastructure, the Parameter group have been removed from all published componentizable Articles of Locations and Creatures variants, as they have been superseded by the new structure (see below)

  • With a revision made to the FrontMatter Parameters infrastructure, all occurrences of Parameter section have been removed as its old functionality is handled by the category Parameter

Hugo Templates-related changes

  • Removed Hugo’s errorf from the Perim Partial Template as it’s not needed anymore

  • Removed HTML markup for OpenLoad and TheVid from Videos Single Page Template as they’re not needed anymore

Changed

Structure-related changes
  • Renamed i18n identifier title-card-search to title-section-card-search, allowing the SEO Partial Template for Article to find a match for the Cards’ Search Tool as well
Hugo Templates-related changes

  • Revamped implementation of the Social Links Partial Template

  • With a revision made to the FrontMatter Parameters infrastructure, both Listing and Single Pages Templates have been updated to reflect the changes in their Section Headers

  • Simplified the way the revamped SEO implementation is included in the Header Partial Template

  • With a revision made to the FrontMatter Parameters infrastructure, the HTML markup of Standalone Pages has been simplified. The <section> tags has its HTML attribute id set with the value of FrontMatter Parameter category and the subcategory Parameter, if defined, is appended its list of classes

  • With the simplification above, the Perim Partial Template used only for Cards’ Articles has also been simplified by removing the redundant Taxonomy Term type from the list of classes wrapping the Content

  • Moved the Page X of Y provided by Hugo’s Paginator from Header Partial Template to Schema.org’s Title Partial Template

  • Simplified (A LOT) the Schema.org’s Title Partial Template, but, most important, we made the title built GLOBAL, by defining it under the scope of passed along by the Partials since the Header Template. This way we can keep everything grouped in one place and, at the same time, use this value to define the <title> tag

  • Updated Schema.org’s BlogPosting (a.k.a Articles) Partial Template to reflect the changes made with the revised FrontMatter Parameters infrastructure and the change of scope of the Title (see above)

  • Moved Twitter OpenGraph tags from Header Partial Template moving to the main SEO Partial Template

  • Moved Yandex Authorization Code up the code along with Facebook’s, this way it also respects the Parameterizable restriction configured by the FrontMatter Parameter analytics

  • Moved OpenGraph tags fb:pages and fb:profile_id from inner SEO Partials to the Common Opengraph Partial

  • With Schema.org’s BreadcrumbList in its own Partial Template, the Breadcrumbs Partial Template has been updated to use the structured provided by the SEO Implementation

  • Universalized the usage of $this while referencing to .this in (almost) all componentizable Partial Templates for SEO, in order to not cause confusion in the future

  • Changed the default value of the Property description for Schema.org’s BlogPosting and TVEpisode (a.k.a Videos). Now, if there’s no Summary defined, the Page Title will be used instead of the Site Name. This prevents N “unpublished” Pages — in the concept of having an actual Article or not — to have the same Description

Public Content-related changes

  • With the renaming of the i18n identifier for Cards’ Search Tool (see above), all occurrences of it have been replaced throughout the site

  • Updated Changelog Page

CSS-related changes

  • Improved the vertical alignment of little arrows near Navigation Menu entries indicating they have sub-menus

  • With a revision made to the FrontMatter Parameters infrastructure, all styles related to the Cards’ search Tool, Terms Page and Changelog Pages have been revised to reflect the changes

Fixed

Structure-related changes

  • Fixed a Markdown syntax for links used in the published Article about Najarin that wasn’t properly interpreted and thus not converted to HTML anchors

  • Fixed a typo in a word in the published Article about Najarin that slipped through FOUR grammar auto-checkers O_O (English only)

  • Fixed, with a palliative solution, an issue with Hugo’s Minifier not properly removing comments in one of the componentizable JS files, thus resulting in syntax errors because of unexpected tokens

Hugo Templates-related changes

  • Fixed an issue with SEO Partial Template for Listings that were using a non-existent i18n identifier when building the headline

  • Fixed an issue with the value of Schema.org’s inLanguage property which had both components of the Language Code separated by an underlines while the right would be a hyphen

  • Fixed an issue with the new componentizable SEO Partial for Webpage in which because a missing value for the isPartOf property, after adding the new componentizable Partial for Breadcrumbs a wrong relation was being created

Public Content-related changes
  • Fixed the Taxonomy Term rarity for Rao’Pa Sahkk Chimegrid, wrongly defined with its initiative

Miscellaneous changes

  • Updated minified CSS and JS
