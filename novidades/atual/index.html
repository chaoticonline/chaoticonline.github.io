Adições

  • Adicionados Arquivos de Conteúdo _index.<LANG>.md sob /changelog a fim de habilitá-lo como uma Seção do Hugo as como parte da nova implementação da Página de Novidades (ver abaixo)

  • Adicionada uma nova possibilidade ao Parâmetro de FrontMatter advertisable nomeada sidebar, permitindo que Páginas exibam blocos de anúncios mas apenas na barra lateral

  • Adicionado um novo Data File para a Página Principal melhorada, mais por causa do path com o logotipo Chaotic (ver abaixo)

  • Adicionado um novo Data File com a estrutura necessária para se traduzir dias da semana e meses como complemento a nova implementação da Página de Novidades (ver abaixo)

  • Adicionadas entradas i18n ausentes para o Artigo componentizável publicado sobre Tangath Toborn, General do Outro Mundo

  • Adicionadas entradas i18n para as novas Redes Sociais como complemento à da nova forma de Compartilhar um Artigo/Página (ver abaixo)

  • Adicionadas entradas i18n para as Seções e Títulos na Página Principal melhorada (ver abaixo)

  • Adicionados ícones SVG para botão de recolhimento do menu pois parece que a nova versão do Bootstrap abandonou suporte ao background reverso (cinza escuro)

  • Adicionado um novo ícone para o menu de Compartilhamento bem como ícones para as outras Redes Sociais como complemento à nova forma de Compartilhar um Artigo/Página (ver abaixo)

  • Adicionado um novo ícone SVG para indicar que um link abrirá em outra aba/janela como complemento à Página Principal melhorada (ver abaixo)

  • Adicionado valor de viewBox ausente ao símbolo SVG do Twitter

Relacionados ao Hugo

  • Implementado uma nova forma de se Compartilhar um Artigo/Página com mais redes Sociais disponíveis

  • Implementado uma forma de se Curtir/Descurtir um Artigo/Página

  • Implementado Preview de Carta na Ferramenta de Busca de Cartas, diretamente de cada resultado encontrado (apenas Criaturas e Locais)

  • Implementada uma forma melhor de lidar com as Revisões de Novidades. Agora em vez de ter todas numa única página, confiando num recurso JS para ocultar grandes revisões, agora ela age como uma Listagem de Resultados, em ordem decrescente e se baseia na marcação especial MORE, mostrando as primeiras alterações de cada revisão e provendo um link para a revisão completa

  • Adicionada uma definição de Paginador alternativa para a Página de Novidades

  • Adicionado, através das Classes Utilitárias do Bootstrap, uma pequena margem superior à Caixa de Informações

Relacionados ao Conteúdo Publicado

  • Adicionado o Parâmetro de FrontMatter publishingDate ausentes nos Artigos publicados sobre o Dractyl e sobre a Cidade Kiru

  • Adicionado o Parâmetro de FrontMatter initiative para ser usado como Termo de Taxonomia em todos os Arquivos de Conteúdo de Locais que não o tinham

  • Adicionada a possibilidade de filtrar algumas páginas específicas dos resultados da Ferramenta de Busca de Cartas.   Atualmente, apenas Rao’Pa Sahkk é ignorado desta forma, porque é um grande Local feito de sub-localizações menores e apenas as menores são Locais reais

Remoções

Estruturais

  • Removido o arquivo CSS externo readmore.css e os arquivos JS externos readmore.js, abbr-touch.js e touchtap-event.js já que não são mais necessários (ver abaixo). Isso, por sua vez, fornecerá uma pequena melhora na performance do tempo de carregamento das Páginas

  • Removidos os Arquivos de Conteúdo changelog.<LANG>.md já que não são mais necessários como parte da nova implementação da Página de Novidades (ver abaixo)

Relacionados ao Hugo

  • Removida a tentativa de carregar os arquivos CSS e JS compilados sem SSL quando executando pelo Servidor do Hugo pois isto acabou não funcionando como esperado quando tunelando

  • Com a nova forma de se Compartilhar um Artigo/Pagina, o <aside> onde os antigos ícones das Redes Sociais costumavam ficar foi removido. Isso, por sua vez, proporcionou mais espaço para o conteúdo principal

  • Removido o bloco de código sobre Datas (criado, publicado e modificado) do Template Parcial de SEO para Listagem de Resultados pois o mesmo código, agora melhorado ainda mais, já estava presente no Template principal

  • Removida a rota alternativa para quando uma Página não tiver descrição (resumo) pois de agora em diante, de um jeito ou de outro, todas elas sempre terão

  • Removida a Classe Utilitária do Bootstrap para fixar uma largura máxima total (w-100) do elemento envolvendo o Bloco de Anúncios do Google AdSense no Template da Página de Vídeos

Alterações

Estruturais

  • Atualizado Bootstrap da versão 4.0.0-alpha.6 para v4.2.1

  • Alterada a forma como a definição expandida dos <abbr> são exibidas em dispositivos móveis e, com essa mudança, o recurso também se tornou disponível para as versões Desktop

  • Alterados os identificadores das CSS Ribbons <_<

  • Página Principal melhorada. Em vez de ter apenas o logotipo grande e centralizado com a headline do site abaixo, agora a Página Principal lista os últimos Artigos publicados e os últimos Vídeos enviados, com miniaturas e camadas de sobreposição com o título das Páginas e, para os mais recentes de cada categoria, destacado com uma miniatura maior, a descrição da Página também

Relacionados ao Hugo

  • Com a atualização feita no Bootstrap, a forma antiga de se ocultar determinadas partes do conteúdo de acordo com viewport mudou e, assim, todas as ocorrências tiveram de ser alteradas ao longo do tema

  • Atualizado o Template Parcial Base para incluir as marcações necessárias para o novo recurso de Gostar/Desgostar e a nova forma de se Compartilhar um Artigo/Página, bem como para substituir <div> genéricas com as tags <header> e <article> mais apropriadas

  • Melhorada a implementação do Template de Listagem de Resultados reestruturando a marcação dos Cards do Bootstrap, o que, por sua vez, melhorou a responsividade da Página

  • Com a nova implementação das Revisões de Novidades, o formato da Data de Publicação foi expandido um pouco para mostrar também o dia da semana traduzível

  • Devolvido o recurso de Fingerprinting dos arquivos compilados e minificados

  • Simplificado a marcação da barra de navegação superior, tanto o contêiner quanto os menus

  • Atualizado o Template Parcial para SEO principal a fim de:

    • Fazer das informações sobre a Imagem Grande e a Descrição das Páginas globais, assim elas podem ser usadas fora dos Templates Parciais de SEO, especificamente como parte da nova forma de se Compartilhar um Artigo/Página
    • Garantir a todas as Páginas, mesmo aquelas que não tenham o devido Parâmetro de FrontMatter, valores pata as propriedades Schema.org dateCreated e datePublished
    • Ter a headline principal como Descrição da Página caso esta não tenha nenhum (sem Resumo), o que é o caso da Página Principal e dos Artigos sem conteúdo real
    • Inverter a condição que incluir os Templates Parciais de temática específica a saber, Artigos ou Vídeos) a fim de poder filtrar a Página Principal, removendo-a, pois como ela não é uma Página Regular para o Hugo e tais Templates Parciais fazem uso de informações exclusivas desse tipo de Página, em vez de simplesmente gerar um erro, o servidor/compilador entra em pânico e aborta tudo o.O

  • Com a alteração feita com respeito ao escopo das informações sobre a Imagem Grande e a Descrição das Páginas, todas as ocorrências ao longo do site foram atualizadas

  • Atualizado o Template Parcial para SEO de Imagem a fim de prover o Logotipo de Chaotic caso nenhum Data File personalizado exista -OU- o path fornecido neles não resolva para uma imagem existente. Dessa forma todas as Páginas agora têm uma Imagem para se criar a devida Entidade do Schema.org

  • Simplificada a marcação da Imagem da Carta no Template de Cartas de Locais

  • Melhorada drasticamente a responsividade das Páginas de Locais

  • Simplificada a marcação do Shortcode Picture, atualmente em uso apenas no Artigo publicado sobre a Caverna de Cristal para exibir a imagem da Cavernas Akkarak

  • Simplificada a condição no Template Parcial para Listagem de Vídeos que distingue os Episódios normais das Curiosidades

  • Simplificada a forma como os Arquivos de Conteúdo tem seus… conteúdos adicionados ao HTML, removendo diversas soluções paliativas às inconsistências de renderização do Hugo
