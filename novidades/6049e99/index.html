Chaotic Novidades rev. 6049e99

Novidades

  rev. 6049e99

 

Adições

Estruturais

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, os Parâmetros listed e comentable que, respectivamente, preveniam um Artigo de ser listado e permitia que o mesmo fosse comentado através do Plugin de Comentários do Facebook, foram adicionados, ambos definidos como false, em todos os Artigos componentizáveis publicados das variantes dos Locais

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, todas as Páginas relacionadas a um nível primário do Menu de Navegação (ex: Perim) receberam em seus arquivos _index.<LANG>.md um novo Parâmetro chamado category

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, todas as Páginas relacionadas a um subnível do Menu de Navegação (ex: Ataques) receberam em seus arquivos /<SUBDIRETÓRIO>/_index.<LANG>.md um novo Parâmetro chamado category, sobrescrevendo aquele de suas páginas-pai -E- um Parâmetro adicional chamado subcategory com o mesmo efeito do, agora removido, Parâmetro group

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, todas as Páginas relacionadas a um nível primário ou a um subnível no Menu de Navegação receberam em seus arquivos _index.<LANG>.md os seguintes Parâmetros:

    • commentable, definindo se tanto a Página de Listagem em si como suas filhas poderão ser discutida por meio do Facebook Comments Plugin
    • analytics, definindo se tanto a Página de Listagem em si como suas filhas receberão o código de coleta de dados do Google Analytics
    • advertisable, definindo se tanto a Página de Listagem em si como suas filhas receberão os blocos de anúncios na barra lateral (visível apenas na versão Desktop) e em meio aos resultados (visível apenas em dispositivos móveis) Teoricamente, esses Parâmetros deveriam ser adicionados apenas nas Páginas de nível mais primário (i.e. /perim/_index.<LANG>.md), mas a disponibilidade deles através da variável .Params do Hugo é muito estrita, com Páginas com cujo pai também é secionado (ex: /criaturas) tendo seus Parâmetros disponíveis através de .CurrentSection.Params. Isso complica o desenvolvimento dos Templates, pois precisaríamos alternar constantemente entre tais aliases

  • Adicionadas duas entradas i18n para distinguir as intenções de Seguir de Compartilhar compartilhar na implementação aprimorada do Template Parcial de Links Sociais

  • Adicionado um Template Parcial para o SiteNavigationElement do Schema.org, para que seja renderizado junto no Template Parcial principal de SEO

  • Adicionado um Template Parcial para o BreadcrumbList do Schema.org, para que seja renderizado junto no Template Parcial principal de SEO

  • Adicionada uma instrução no arquivo config.yaml para remover automaticamente acentos dos paths, o que permitiu uma redução maciça na implementação melhorada de SEO para Vídeos, já que não precisamos mais removê-los manualmente

  • Adicionado um identificador i18n para os Termos de Uso, permitindo que o Template Parcial para Artigos encontre uma combinação para tal Página também

Relacionados ao Hugo
  • Adicionadas as Propriedades isPartOf e mainEntityOfPage no Template Parcial de SEO para Lists
Relacionados ao Conteúdo Publicado

  • Adicionado Termo de Taxonomia ausente type para a Página do Oásis de Mipedim (apenas em Inglês)

  • Adicionado Termo de Taxonomia ausente initiative para as Páginas do Tanque de Lava e Carnaval da Confusão (apenas Português Brasileiro)

  • Adicionada uma versão reduzida da introdução da Página da Ferramenta de Busca de Cartas como Resumo, assim a implementação SEO tem algo melhor para com o que trabalhar

  • Publicado Episódio #22, Temporada #01 com áudio em Português Brasileiro

Remoções

Estruturais

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, o Parâmetro group foi removido de todos os Artigos componentizáveis publicados sobre as variantes de Locais e Criaturas, já que este foi suplantado pela nova estrutura (ver abaixo)

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, todas as ocorrências do Parâmetro section foram removidas já que sua funcionalidade agora é gerenciada pelo Parâmetro category

Relacionados ao Hugo

  • Removido errorf do Hugo do Template Parcial Perim já que este não é mais necessário

  • Removidas as marcações referentes ao OpenLoad e TheVid do Template da Página individual de Vídeos já que eles não são mais usados

Alterações

Estruturais
  • Renomeado o identificador i18n title-card-search para title-section-card-search, permitindo que o Template Parcial para SEO de Artigos encontre uma ocorrência para a Ferramenta de Busca de Cartas também
Relacionados ao Hugo

  • Melhorada a implementação do Template Parcial de Links Sociais

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, ambas os Templates de Páginas de Listagem e Páginas Individuais foram atualizados para refletir as alterações feitas em seus Cabeçalhos de Seção

  • Simplificada a forma como a implementação melhorada de SEO é incluída no Template Parcial de Cabeçalho

  • Após a revisão feita à infraestrutura dos Parâmetros de FrontMatter, a marcação HTML das Páginas Individuais foi simplificada. As tags <section> agora têm o atributo HTML id configurado com o valor do Parâmetro category e o Parâmetro subcategory, se definido, é adicionado ao fim da lista de classes

  • Com a simplificação acima, a marcação HTML do Template Parcial Perim usado apenas nos Artigos das Cartas também foi simplificado, removendo o termo de Taxonomia type redundante da lista de classes do elemento que envolve o Conteúdo

  • Movido o Página X de Y fornecido pelo Paginator do Hugo do Template Parcial de Cabeçalho para o Template Parcial do Schema.org de Título

  • Simplificado (E BASTANTE) o Template Parcial do Schema.org de Título, mas, mais importante, fizemos com que o título construído se tornasse GLOBAL, definindo-o sob o escopo passado pelos Templates Parciais desde o Cabeçalho. Dessa forma podemos manter tudo agrupado em um único lugar e, ao mesmo tempo, usar o valor para criar a tag <title>

  • Atualizado o Template Parcial do Schema.org de BlogPosting (“Artigos”) para refletir as mudanças feitas na infraestrutura revisada dos Parâmetros de FrontMatter e a mudança de escopo do Título (ver acima)

  • Movido as tags OpenGraph do Twitter do Template Parcial de Cabeçalho para o Template Parcial principal de SEO

  • Movido o código de autorização Yandex mais acima junto ao do Facebook, assim este também respeita a Parametrização configurada pelo Parâmetro de FrontMatter analytics

  • Movidas as tags OpenGraph fb:pages e fb:profile_id dos Templates Parciais de SEO mais internos para o Template de Tags OpenGraph Comuns

  • Com a Entidade BreadcrumbList do Schema.org em seu próprio Template Parcial, o Template dos Breadcrumbs foi atualizado para usar a estrutura fornecida por esta implementação SEO

  • Universalizado uso da variável de Template $this quando se referenciado ao .this em (quase) todos os Templates Parciais para SEO componentizáveis, a fim de não causar confusão no futuro

  • Alterada o valor-padrão da Propriedade description das Entidades Schema.org para BlogPosting e TVEpisode (a.k.a. Vídeos). Agora, se não houver um Resumo definido, o Título da Página será usado em vez do Nome do Site. Isso previne diversas Páginas “não publicadas” — no conceito de se ter ou não um Artigo de verdade — terem a mesma Descrição
