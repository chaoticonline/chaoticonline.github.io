Adições

Estruturais

  • Adicionada entrada i18n ausente para Tangath Toborn, Em Treinamento

  • Adicionado um Arquivo de Conteúdo para facilitar o desenvolvimento da Página 404, embora, por enquanto, após compilado, o arquivo resultante deva ser manualmente renomeado e movido um nível acima

  • Adicionado o projeto Hugo Debugprint ao tema principal, condicionando seu CSS ao ambiente de desenvolvimento

Remoções

Estruturais

  • Com a alteração na estrutura dos Arquivos de Conteúdo (ver abaixo), todos os arquivos localizados em /content foram movidos um nível acima de acordo com o idioma definido, o que, de qualquer jeito, significa a remoção e a adição de arquivos

  • Removidos os Parâmetros de FrontMatter target e language já que não são mais necessários

Relacionados aos Estilos do Site
  • Removido o brilho ao se clicar nos menus dropdown na Página da Ferramenta de Busca de Cartas

Alterações

Estruturais

  • Refeita TODA a estrutura de Arquivos de Conteúdo seguindo o modelo multilíngue /content/<LANG>/<SEÇÃO>/arquivo.md assim como os Data Files, entradas i18n e arquivos estáticos específicos de um idioma, o que, de qualquer jeito, significa a remoção e a adição de arquivos

  • Com a reestruturação feita acima, TODOS os Arquivos de Conteúdo foram atualizados para:

    1. Usarem um slug em vez de uma URL personalizada Essa alteração se deu pois o que antes URLs pequenas era preferíveis, hoje URLs verbosas e bem descritivas parecem ser aquilo que se espera de um site <_<
    2. Criar uma relação entre Arquivos de Conteúdo através de múltiplos idiomas através do Parâmetro de FrontMatter translationKey pré-definido pelo Hugo, pois quando algum diretório entre /content e o nome do arquivo divergir da estrutura do idioma padrão (Inglês) o relacionamento se perde e o Hugo não pode sincronizar as traduções “automagicamente”
    3. Atualizar links em TODOS os Artigos publicados e Página dos Episódios
    4. Remover Parâmetros de FrontMatter desnecessários (ver acima)

  • Com a inclusão do Arquivo de Conteúdo para desenvolvimento da Página de 404 (ver acima), um arquivo .gitignore foi adicionado para negar o diretório /404 de ser versionado (e publicado no GitHub Pages)

  • Refeita a forma como o menu de navegação é gerado, construindo tudo no config.yaml em vez de listar cada entrada manualmente

Relacionados ao Hugo

  • Melhorada a solução paliativa para se ter todos os Cards do Bootstrap com o mesmo tamanho

  • Alterada a forma como o Paginator do Hugo é configurado diferentemente entre a Página de Novidades de todas as outras Seções do site pois com a nova estrutura multilíngue, por algum motivo, confiar em .Section para tal distinção não parece dar certo, mesmo que expandamos a condição para levar em conta todos os idiomas existentes

  • Alterado o Template de Termos de Taxonomia para não incluir na listagem a Página Principal de Novidades pois com a nova estrutura multilíngue esta Página, em todas as localizações faz uso do Parâmetro de FrontMatter type do Hugo e esta expressão conflita o nosso Termo de Taxonomia type reservado para o Tipo das Cartas

  • Melhorado o Template de Termos de Taxonomia, não apenas para que ele seguisse o “modelo” dos outros Templates por fazer uso do Template Base, mas também para que nenhum Bloco de Anúncio aparecesse nele, indiretamente melhorando a performance da Ferramenta de Busca de Cartas

Relacionados ao Conteúdo Publicado

  • Melhorada a exibição da Imagem Grande do Targubaj, focando mais no topo do que na base

  • Agora que quaisquer observações a respeito de um ou mais Dubladores de uma Criatura quebra para uma nova linha, foi possível complementar as observações sobre o Cerbie, indicando em seu Data File que Esquerda, Meio e Direita se referem às suas cabeças (apenas Português Brasileiro)

  • Atualizada Página de Novidades

Relacionados aos Estilos do Site

  • Melhorado o posicionamento dos Autores de uma Citação

  • Melhoradas as margens na Página de Termos de Taxonomia

  • Melhorado alinhamento vertical do texto nos menus dropdown na Página da Ferramenta de Busca de Cartas
